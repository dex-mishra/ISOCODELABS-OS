Composition

Composition determines how visual elements are arranged within the layout.

While the layout establishes the structural framework of a page, composition determines how attention moves through that framework.

Good composition feels effortless.

Users should never consciously analyze why one interface feels easier to understand than another.

Instead, attention should naturally flow toward the information that matters most.

Composition is the discipline of intentionally directing attention.

Every visual decision should support this objective.

⸻

Composition Philosophy

Composition is not decoration.

Composition is visual communication.

Every element on a screen competes for attention.

The designer’s responsibility is to decide which elements deserve that attention.

Users should never be forced to determine importance for themselves.

The interface should make that decision obvious.

CP-007

never mathematically symmetrical by default.

⸻

CP-008

CP-009

Users should always know where to look next.

⸻

CP-010

Every viewport should communicate one primary message.

Visual Hierarchy

Visual hierarchy determines the order in which information is discovered.

Hierarchy should remain understandable before color, animation or illustration are introduced.

⸻

CP-011

Hierarchy should primarily be established through:

1. Position
2. Scale
3. Whitespace
4. Contrast
5. Motion
6. Color

⸻

CP-012

Do not rely on color as the primary hierarchy mechanism.

⸻

CP-013

The primary message should be immediately recognizable.

⸻

CP-014

Supporting information should never compete with primary information.

⸻

CP-015

Reduce emphasis before adding emphasis elsewhere.

⸻

CP-016

Large typography should remain meaningful.

⸻

CP-017

Avoid assigning equal visual weight to unrelated content.

⸻

CP-018

Every increase in emphasis should correspond to an increase in importance.

⸻

CP-019

Hierarchy should remain consistent throughout the product.

⸻

CP-020

Visual importance should accurately reflect business importance.

⸻

Rhythm

Rhythm controls the pacing of visual discovery.

Interfaces should feel composed,

not repetitive.

⸻

CP-021

Alternate dense regions with spacious regions.

⸻

CP-022

Avoid identical section structures repeating excessively.

⸻

CP-023

Create intentional moments of visual silence.

⸻

CP-024

Major content blocks deserve breathing room before and after.

⸻

CP-025

Avoid uninterrupted sequences of visually similar components.

⸻

CP-026

Use variation to reinforce narrative,

not novelty.

⸻

CP-027

Users should experience a sense of progression while scrolling.

⸻

CP-028

Every major reveal should have appropriate anticipation.

⸻

CP-029

The visual rhythm should support the storytelling rhythm.

⸻

CP-030

Scrolling should feel like unfolding a narrative rather than reading a checklist.

⸻

Balance

Balance communicates confidence.

An unbalanced interface feels accidental.

A perfectly symmetrical interface often feels lifeless.

The objective is intentional equilibrium.

⸻

CP-031

Balance should emerge through perceived visual weight,

not identical dimensions.

⸻

CP-032

Large empty regions require justification.

⸻

CP-033

Dense regions should be counterbalanced by visual breathing space.

⸻

CP-034

Avoid visual heaviness accumulating on one side of the viewport.

⸻

CP-035

Visual tension should be introduced intentionally.

⸻

CP-036

Objects should appear physically stable.

⸻

CP-037

Large media elements require sufficient surrounding whitespace.

⸻

CP-038

Decorative graphics should not disturb visual balance.

⸻

CP-039

The interface should feel structurally grounded.

⸻

CP-040

Users should subconsciously perceive order.

⸻

Reading Flow

Interfaces are experienced as journeys,

not screenshots.

Reading flow should anticipate user behavior.

⸻

CP-041

Every viewport should naturally encourage the next interaction.

⸻

CP-042

Headlines should create curiosity.

Supporting content should satisfy it.

⸻

CP-043

Avoid abrupt interruptions in reading flow.

⸻

CP-044

Long-form pages should periodically reward continued exploration.

⸻

CP-045

Calls-to-action should emerge naturally from understanding.

⸻

CP-046

Do not interrupt educational content with premature conversion requests.

⸻

CP-047

Every section should answer a question introduced previously.

⸻

CP-048

Avoid visual dead ends.

⸻

CP-049

The user’s attention should rarely reverse direction.

⸻

CP-050

Composition should encourage continuous progression.

⸻

Layering

Layering establishes spatial relationships between interface elements.

It should communicate hierarchy,

not visual complexity.

⸻

CP-051

Every layer should have a defined purpose.

⸻

CP-052

Avoid unnecessary overlapping elements.

⸻

CP-053

Foreground elements should remain visually distinct.

⸻

CP-054

Background layers should support,

not compete.

⸻

CP-055

Depth should communicate structure before aesthetics.

⸻

CP-056

Avoid excessive floating panels.

⸻

CP-057

Layer transitions should remain predictable.

⸻

CP-058

Objects appearing closer to the user should generally require greater interaction priority.

⸻

CP-059

Visual depth should remain physically believable.

⸻

CP-060

Layering should simplify understanding,

not complicate it.

Self Review

Before implementation verify:

✓ One dominant focal point

✓ Clear hierarchy

✓ Balanced composition

✓ Comfortable visual rhythm

✓ Progressive reading flow

✓ Intentional layering

✓ Minimal competing elements

✓ Strong narrative progression

✓ Comfortable scanning

✓ Composition remains understandable without color or animation

If users instinctively know where to look, what to read and what to do next without conscious effort,

the composition is successful.
