🔗 API Integration Setup Guides

Follow these guides to connect all external services. Complete these BEFORE running the app.

1. Google Cloud Platform (Meet, Gmail, Vertex AI)

  • Go to console.cloud.google.com → Create a new project (or use existing)
  • Enable APIs: Google Calendar API, Gmail API, Vertex AI API
  • Go to "APIs & Services" → "Credentials" → Create OAuth 2.0 Client ID
  • Application type: Web application
  • Authorized redirect URIs: http://localhost:3000/api/auth/google/callback (dev) and your production URL
  • Copy Client ID → GOOGLE_CLIENT_ID in .env
  • Copy Client Secret → GOOGLE_CLIENT_SECRET in .env
  • Set GOOGLE_REDIRECT_URI=http://localhost:3000/api/auth/google/callback
  • For Vertex AI: GCP_PROJECT_ID=your-project-id, GCP_LOCATION=us-central1
  • Ensure the service account has "Vertex AI User" role
  • OAuth consent screen: Add scopes calendar.events and gmail.readonly

2. WhatsApp Business API (Meta Cloud API)

  • Go to developers.facebook.com → Create a Meta Business App
  • Add "WhatsApp" product to your app
  • Go to WhatsApp → "API Setup" → Get temporary access token (for dev)
  • For production: Create a System User → Generate permanent token
  • Note your Phone Number ID from the dashboard
  • Set in .env: WHATSAPP_API_TOKEN=your_token, WHATSAPP_PHONE_NUMBER_ID=your_phone_id
  • Configure webhook URL: https://your-domain.com/api/webhooks/whatsapp
  • Webhook verify token: any string you choose (store in env)
  • Subscribe to: messages, message_deliveries

3. Nanobana API (Image Generation)

  • Sign up at the Nanobana platform
  • Navigate to API section → Generate API key
  • Set in .env: NANOBANA_API_KEY=your_api_key
  • Test via Settings → Integrations → "Test Connection" button
  • Rate limits: Check your plan tier for requests per minute
  • Supported parameters: style, resolution, aspect ratio (check their docs)

4. Veo API (Video Generation)

  • Veo uses the same GCP project as Vertex AI
  • Ensure "Vertex AI API" is enabled (covers Veo access)
  • Your GCP service account needs "Vertex AI User" role (same as step 1)
  • Set VEO_API_KEY in .env OR it uses default GCP credentials
  • Video generation takes 30-120 seconds — the app handles this async
  • Output stored in GCS bucket (configure GCS_BUCKET_NAME if needed)

5. Fathom AI (Meeting Recording)

  • No API setup needed — Fathom uses shared links
  • After each meeting, Fathom provides a share link like: https://fathom.video/share/abc123
  • Users paste this link in the meeting detail page
  • The summary and transcript are entered manually from the Fathom shared page
  • Ensure team members have Fathom installed and meetings are being recorded

6. Database (PostgreSQL)

  • Local dev: Install PostgreSQL or use Docker: docker run -p 5432:5432 -e POSTGRES_PASSWORD=postgres postgres
  • Create database: createdb isocodelabs_ops
  • Set DATABASE_URL=postgresql://user:password@localhost:5432/isocodelabs_ops
  • Production (GCP): Use Cloud SQL for PostgreSQL — create instance, get connection string
  • Run migrations: npx prisma migrate deploy
  • Seed data: npx prisma db seed

7. Redis (Caching + Real-time Pub/Sub)

  • Local dev: Install Redis or Docker: docker run -p 6379:6379 redis
  • Set REDIS_URL=redis://localhost:6379
  • Production (GCP): Use Memorystore for Redis
  • Used for: session cache, dashboard metrics cache (5-min TTL), WebSocket pub/sub
